# Документация по фронтенду приложения Kanban

## Обзор проекта

Проект представляет собой фронтенд для системы Kanban. В нем реализованы различные компоненты для работы с проектами и задачами, а также механизмы входа и регистрации пользователей. Вся структура разделена на отдельные компоненты, с использованием стилей и функций для взаимодействия с сервером через API.

---

## Структура проекта

```plaintext

  ├── index.js
```

## Основные компоненты

### `Header.js`

**Описание**: Этот компонент отвечает за отображение верхней части страницы, включая логотип и кнопки для входа, регистрации, перехода к проектам и задачам.

**Основные функции**:
- Отображение логотипа и приветствия.
- Обработка кликов на кнопки, такие как "Проекты" и "Задачи".
- Проверка на наличие пользователя (если пользователь авторизован, отображается информация о нем).

**Связь с другими компонентами**:
- Вызывает окна входа и регистрации (`LoginForm` и `RegistrationForm`).
- Передает данные о текущем пользователе в другие компоненты.

### `KanbanBoard.js`

**Описание**: Компонент для отображения канбан-доски, где задачи могут перетаскиваться между колонками (статусами).

**Основные функции**:
- Отображение задач в 3 статусах: "To Do", "In Progress", "Done".
- Реализация drag-and-drop функционала с использованием библиотеки `@hello-pangea/dnd`.
- Обновление статуса задач при перетаскивании и отправка данных на сервер для сохранения изменений.

**Связь с другими компонентами**:
- Получает задачи из родительского компонента (`App.js`).
- Обновляет задачи через родительский компонент, отправляя изменения на сервер.

### `LoginForm.js`

**Описание**: Компонент формы входа, который позволяет пользователям авторизоваться в системе.

**Основные функции**:
- Принимает данные пользователя (email и пароль) и отправляет их на сервер для проверки.
- В случае успешного входа сохраняет данные о пользователе в `localStorage`.

**Связь с другими компонентами**:
- Передает данные о пользователе в родительский компонент (`App.js`) через функцию `onLoginSuccess`.

### `RegistrationForm.js`

**Описание**: Компонент формы регистрации, который позволяет пользователям зарегистрироваться в системе.

**Основные функции**:
- Принимает данные пользователя (email и пароль) и отправляет их на сервер для регистрации.

**Связь с другими компонентами**:
- При успешной регистрации вызывает родительскую функцию для закрытия окна и перехода к форме входа.

### `TaskForm.js`

**Описание**: Компонент формы добавления задач. Он позволяет пользователю добавлять задачи в выбранный проект.

**Основные функции**:
- Принимает данные задачи и отправляет их на сервер для добавления.

**Связь с другими компонентами**:
- Передает данные задачи в родительский компонент для обновления списка задач.

### `MainContent.js`

**Описание**: Компонент, отвечающий за основное содержимое страницы.

**Основные функции**:
- Отображает список проектов и задачи, привязанные к выбранному проекту.
- Обрабатывает добавление задач через `TaskForm` и отображение задач через `TaskList` и `KanbanBoard`.

**Связь с другими компонентами**:
- Получает и передает данные о задачах и проектах в другие компоненты, такие как `Projects`, `TaskForm`, и `KanbanBoard`.

### `ProjectList.js`

**Описание**: Этот компонент отображает список всех проектов и позволяет пользователю выбрать проект для работы с задачами.

**Основные функции**:
- Отображение списка проектов и возможность выбрать проект.

## API

### `TaskApi.js`

**Описание**: Файл с функциями для работы с задачами через API. Это включает в себя такие операции, как:
- Получение списка задач.
- Добавление новых задач.
- Обновление статуса задач.

### `ProjectApi.js`

**Описание**: Файл с функциями для работы с проектами через API. Основные функции:
- Получение списка проектов.

### `UserApi.js`

**Описание**: Файл с функциями для работы с пользователями через API:
- Получение данных пользователя по ID.
- Регистрация нового пользователя.
- Авторизация.

---

## Основные взаимосвязи между компонентами

1. **`App.js`**: Основной компонент приложения, который управляет состоянием задач, выбранного проекта и авторизацией пользователя. Он также обрабатывает логику открытия окон входа и регистрации. `App.js` передает данные в дочерние компоненты, такие как `Header`, `MainContent`, и т. д.

2. **`Header.js`**: Отвечает за отображение логотипа и кнопок, связанных с авторизацией и навигацией. Он также взаимодействует с окнами входа и регистрации, передавая информацию о текущем пользователе.

3. **`MainContent.js`**: Отображает основное содержимое в зависимости от выбранного проекта и задач. Компоненты внутри него, такие как `KanbanBoard` и `TaskForm`, управляют данными о задачах и отображают их на странице.

4. **`KanbanBoard.js`**: Основной компонент для отображения задач в виде канбан-доски. Он использует drag-and-drop для перемещения задач между статусами, отправляя обновления на сервер.

5. **`LoginForm.js` и `RegistrationForm.js`**: Обрабатывают формы входа и регистрации, взаимодействуя с API для авторизации и регистрации пользователей.

---

## Заключение

Этот фронтенд реализует простую канбан-доску, в которой пользователи могут управлять задачами в разных статусах. Взаимодействие между компонентами происходит через пропсы и события, а также через API-запросы для работы с данными. Компоненты разделены по логическим частям, что позволяет легко расширять и поддерживать проект.
